# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

**All Tasks (Home)**
- **Text input:** Type a task title into the "What needs to be done?" field.
- **Add button:** Click to add the task to the list. The input clears automatically after adding.
- **Task checkbox:** Click the checkbox next to a task to mark it as done (or to undo it).
- **Delete button (✕):** Click the X on the right side of a task to permanently remove it.
- **Stats bar:** Displays the current count of Total, Active, and Done tasks, updating automatically as you change them.

**Active Tasks**
- Shows only the tasks that are not yet completed. You can still add new tasks, mark them done (which removes them from this view), or delete them.

**Done Tasks**
- Shows only the tasks that have been marked as completed. You can uncheck a task to move it back to active, or delete it.

**Navigation**
- **Top bar links:** Click **All**, **Active**, or **Done** in the top navigation to switch between the filtered views. The currently active view is highlighted.

## Quick test

1. Open the app and confirm the "All Tasks" view loads with the top navigation showing **All**, **Active**, and **Done** links.
2. Type a task name into the input field and click **Add**. Verify the task appears in the list with an "Active" badge and a random avatar image.
3. Click the task's checkbox. Confirm the task title appears crossed out, the badge changes to "Done", and the "Active" and "Done" numbers in the stats bar update.
4. Click **Done** in the top navigation. Confirm only the completed task is visible. Click **Active** and confirm only uncompleted tasks are visible.
5. Click the **✕** (delete) button on a task. Confirm the task is removed from the list and the stats bar numbers decrease.
6. If the app is configured with a backend (`window.API_BASE`), open your browser's Network tab, add a new task, and confirm a `POST` request to `/api/store/tasks` returns a successful response.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
