# Quick To-Do

A simple to-do list app: add a task with a title, mark tasks as done, delete tasks, and filter by All / Active / Done.

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

### Home Page
This is the only screen in the app, where you manage your entire to-do list.

- **Task Input**: Type a task title into the "What needs to be done?" text box and click **Add** (or press Enter) to create a new task.
- **Filter Bar**: Click **All**, **Active**, or **Done** to filter the list. Each button shows a count pill indicating how many tasks match that category.
- **Task Item**: Each task displays its title, a small avatar icon, and action buttons:
  - **✓ Done**: Click this to mark a task as complete. The title gets a line-through, a "Done" badge appears, and the button changes to **↩ Undo**.
  - **↩ Undo**: Click this to mark a completed task as active again.
  - **✕**: Click this to delete the task from the list entirely.
- **Empty State**: If the current filter has no tasks, you will see the message "No tasks here. Add one above!"

## Quick test

1. Open the app and confirm the "Quick To-Do" heading, the task input field, and the filter buttons (All, Active, Done) are visible.
2. Type a task name into the input and click **Add**; verify the task appears in the list below.
3. Click **✓ Done** on the new task; confirm the title has a line-through, a "Done" badge appears, and the button changes to **↩ Undo**.
4. Click the **Active** filter and verify the completed task disappears; click **Done** and verify only the completed task shows; click **All** to see every task.
5. Click **↩ Undo** on the completed task to change it back to active, then click **✕** to delete it; confirm it vanishes from the list.
6. *(If a backend is configured)* Open your browser's Network tab, add a new task, and confirm a `POST` request to `/api/store/tasks` returns a successful response.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
